# Level 5 to 7 Roadmap

These levels extend the original training toward more advanced PL/SQL work.

## Level 5: Collections and Bulk Processing

- `33` records
- `34` nested tables
- `35` `BULK COLLECT`
- `36` `FORALL`

Outcome:

- You can model rows in memory.
- You understand how to fetch and move many rows efficiently.

## Level 6: Controlled Data Access

- `37` explicit cursors
- `38` parameterized cursors
- `39` dynamic SQL
- `40` user-defined exceptions

Outcome:

- You can write procedures that adapt to inputs and handle failures intentionally.

## Level 7: Advanced PL/SQL Patterns

- `41` package state
- `42` package-returned collections
- `43` autonomous transaction logging
- `44` compound triggers

Outcome:

- You start using patterns that look much more like real Oracle application code.

## What "Advanced" Means After These Levels

If you complete Levels 5 to 7 comfortably, you should be able to:

- design small packages on your own
- choose between loops, cursors, and bulk operations
- write safer exception handling
- understand transaction boundaries more clearly
- read more advanced Oracle code without getting lost immediately

You would still have room to grow in:

- performance tuning with execution plans
- advanced SQL optimization
- enterprise package architecture
- secure dynamic SQL design
- deployment and migration workflows
